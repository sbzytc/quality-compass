# خطة نظام "رصدة" للعيادات والمستشفيات

## نظرة عامة

هذا المستند يوضح خطة تحويل نظام "رصدة" من نظام مراقبة جودة الفروع إلى نظام متخصص للعيادات والمستشفيات.

**الهيكل الجديد:** `المستشفى/المجمع ← الأقسام ← الغرف`

---

## 🏥 أنواع الغرف والأقسام (Room Types)

### الأقسام الرئيسية (Departments)
| القسم | الوصف |
|-------|-------|
| `قسم العمليات` | غرف العمليات والتعقيم |
| `قسم العيادات` | العيادات التخصصية |
| `قسم الطوارئ` | الطوارئ والإسعاف |
| `قسم الإستقبال` | الاستقبال والتسجيل |
| `قسم المختبر` | التحاليل والأشعة |
| `قسم الصيدلية` | الصيدلية الداخلية |
| `قسم التمريض` | محطات التمريض |
| `قسم النظافة` | التنظيف والتعقيم العام |

## 🗄️ تعديلات قاعدة البيانات

### الجداول الجديدة

#### 1. جدول الأقسام (`departments`)
```sql
CREATE TABLE public.departments (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  branch_id UUID REFERENCES public.branches(id) ON DELETE CASCADE NOT NULL,
  name VARCHAR(255) NOT NULL,                    -- اسم القسم
  name_ar VARCHAR(255),                          -- الاسم بالعربية
  department_type VARCHAR(50) NOT NULL,            -- نوع القسم
  code VARCHAR(50),                                -- كود القسم
  is_active BOOLEAN DEFAULT true,
  manager_id UUID,                               -- مدير القسم
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### 2. جدول الغرف (`rooms`)
```sql
CREATE TABLE public.rooms (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  department_id UUID REFERENCES public.departments(id) ON DELETE CASCADE NOT NULL,
  branch_id UUID REFERENCES public.branches(id) ON DELETE CASCADE NOT NULL,
  name VARCHAR(255) NOT NULL,                    -- اسم الغرفة
  name_ar VARCHAR(255),                          -- الاسم بالعربية
  room_type VARCHAR(50) NOT NULL,                -- نوع الغرفة (enum)
  room_number VARCHAR(50),                         -- رقم الغرفة
  floor_number INTEGER,                            -- رقم الطابق
  capacity INTEGER DEFAULT 1,                      -- السعة
  equipment JSONB,                                 -- المعدات (JSON)
  is_active BOOLEAN DEFAULT true,
  last_evaluation_date DATE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Index للبحث السريع
CREATE INDEX idx_rooms_branch ON public.rooms(branch_id);
CREATE INDEX idx_rooms_department ON public.rooms(department_id);
CREATE INDEX idx_rooms_type ON public.rooms(room_type);
```

#### 3. جدول معدات الغرفة (`room_equipment`)
```sql
CREATE TABLE public.room_equipment (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  room_id UUID REFERENCES public.rooms(id) ON DELETE CASCADE NOT NULL,
  equipment_name VARCHAR(255) NOT NULL,          -- اسم الجهاز/المعدة
  equipment_type VARCHAR(100),                     -- نوع الجهاز
  serial_number VARCHAR(100),                      -- الرقم التسلسلي
  manufacturer VARCHAR(255),                       -- الشركة المصنعة
  last_maintenance_date DATE,                    -- آخر صيانة
  next_maintenance_date DATE,                    -- الصيانة القادمة
  is_functional BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### تعديلات على الجداول الحالية

#### تعديل `evaluation_templates`
```sql
-- إضافة حقل لربط القالب بنوع الغرفة
ALTER TABLE public.evaluation_templates 
  ADD COLUMN room_type VARCHAR(50) NULL,
  ADD COLUMN template_category VARCHAR(50) DEFAULT 'branch'; -- 'branch' | 'room' | 'department'

-- Index
CREATE INDEX idx_evaluation_templates_room_type ON public.evaluation_templates(room_type);
```

#### تعديل `evaluations`
```sql
-- إضافة حقب للتقييم على مستوى الغرفة
ALTER TABLE public.evaluations 
  ADD COLUMN room_id UUID NULL REFERENCES public.rooms(id),
  ADD COLUMN department_id UUID NULL REFERENCES public.departments(id),
  ADD COLUMN evaluation_scope VARCHAR(20) DEFAULT 'branch'; -- 'branch' | 'room' | 'department'

-- Index
CREATE INDEX idx_evaluations_room ON public.evaluations(room_id);
CREATE INDEX idx_evaluations_department ON public.evaluations(department_id);
```

#### تعديل `non_conformities`
```sql
-- إضافة ربط الملاحظات بالغرفة
ALTER TABLE public.non_conformities 
  ADD COLUMN room_id UUID NULL REFERENCES public.rooms(id),
  ADD COLUMN department_id UUID NULL REFERENCES public.departments(id);

-- Index
CREATE INDEX idx_non_conformities_room ON public.non_conformities(room_id);
```

### 3. قالب غرفة الإفاقة (Recovery Room)

**أ) مراقبة ما بعد التخدير (40%)**
- مقياس مستوى الوعي والتخدير
- مراقبة العلامات الحيوية المستمرة
- توفر أكسجين وشفط
- توفر معدات معالجة التقيؤ
- قرب من طبيب التخدير

**ب) السلامة والراحة (30%)**
- أسرة قابلة للتعديل
- قيود واقية للمريض
- إضاءة مناسبة
- درجة حرارة مريحة
- هدوء وخصوصية

**ج) المعدات الطارئة (30%)**
- قرب من عربة الطوارئ
- أدوات تنبيب احتياطية
- أدوات التنفس الاصطناعي
- أدوات إحياء القلب

---

### 4. قالب الاستقبال (Reception)

**أ) خدمة العملاء (35%)**
- سرعة استقبال المريض (< 5 دقائق)
- لباقة واحترافية الموظفين
- وضوح الإجراءات والتعليمات
- توفر خدمة الترجمة إن لزم

**ب) الكفاءة التشغيلية (35%)**
- نظام تسجيل إلكتروني فعال
- دقة بيانات المريض
- نظام تحويل سلس للعيادات
- توفر المعلومات المطلوبة

**ج) المظهر والتنظيم (30%)**
- نظافة وترتيب المنطقة
- وضوح اللوحات الإرشادية
- راحة مقاعد الانتظار
- توفر المعلومات الصحية

---

### 5. قالب غرفة الانتظار (Waiting Room)

**أ) الراحة والنظافة (40%)**
- نظافة الأرضيات والمقاعد
- راحة المقاعد وتوزيعها
- تهوية وإضاءة مناسبة
- درجة حرارة مريحة

**ب) الخدمات والتجهيزات (30%)**
- توفر ماء للشرب
- دورات مياه نظيفة
- منطقة أطفال/ترفيه
- شاشة عرض للمواعيد

**ج) التنظيم والسير (30%)**
- تنظيم الدخول والخروج
- الفصل بين الجنسين إن لزم
- وضوح الإرشادات
- سهولة الوصول للمقعدين

---

### 6. قابل المختبر (Laboratory)

**أ) جودة العينات (35%)**
- إجراءات أخذ العينات السليمة
- التعريف الصحيح للعينات
- التخزين المناسب قبل التحليل
- التعامل مع العينات المعدية

**ب) المعدات والدقة (35%)**
- معايرة الأجهزة الدورية
- جودة الاختبارات الداخلية/خارجية
- صيانة الأجهزة المجدولة
- التحكم في درجة الحرارة

**ج) التوثيق والأمان (30%)**
- سرية نتائج المرضى
- دقة التقارير والتوثيق
- نظام التتبع والمراجعة
- التخلص الآمن من العينات

---

### 7. قالب الصيدلية (Pharmacy)

**أ) التخزين والأمان (35%)**
- تخزين الأدوية حسب الحرارة/الرطوبة
- فصل الأدوية المراقبة
- نظام FIFO للانتهاء
- تأمين الأدوية الخطرة

**ب) الكفاءة والدقة (35%)**
- دقة الروشتات والجرعات
- فحص التداخلات الدوائية
- وضوح تعليمات الاستخدام
- سرعة تجهيز الأدوية

**ج) الخدمة والنظافة (30%)**
- نظافة منطقة الصيدلية
- خصوصية استشارة المريض
- توفر الأدوية الأساسية
- التعامل مع الشكاوى الدوائية

```
                                                              │
                         ┌──────────────────────────────────────┘

              ┌──────────────────────┐
              │ إجراء التقييم        │

              └──────────────────────┘
                         │
                         ▼
              ┌──────────────────────┐

              │ تلقائية (<=3)       │
              └──────────────────────┘

              └──────────────────────┘
```

### 3. متوسط أداء المستشفى

```
متوسط المستشفى = (مجموع درجات جميع الغرف) / (عدد الغرف المقيمة)

معاملات الوزن:
- غرفة العمليات: 1.5 (أهمية عالية)
- العيادات: 1.2
- الطوارئ: 1.3
- الباقي: 1.0
```

## 🔐 سياسات الأمان (RLS Policies)

```sql
-- departments RLS
CREATE POLICY "Branch managers can manage their departments"
ON public.departments FOR ALL
TO authenticated
USING (
  branch_id IN (
    SELECT branch_id FROM public.profiles 
    WHERE user_id = auth.uid() AND branch_id IS NOT NULL
  )
  OR EXISTS (
    SELECT 1 FROM public.user_roles 
    WHERE user_id = auth.uid() AND role IN ('admin', 'executive')
  )
);

-- rooms RLS
CREATE POLICY "Users can view rooms in their scope"
ON public.rooms FOR SELECT
TO authenticated
USING (
  branch_id IN (
    SELECT branch_id FROM public.profiles WHERE user_id = auth.uid()
    UNION
    SELECT id FROM public.branches WHERE region_id IN (
      SELECT region_id FROM public.profiles WHERE user_id = auth.uid() AND region_id IS NOT NULL
    )
  )
  OR EXISTS (
    SELECT 1 FROM public.user_roles 
    WHERE user_id = auth.uid() AND role IN ('admin', 'executive')
  )
);
```
